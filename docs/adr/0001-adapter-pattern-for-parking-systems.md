# ADR-0001: 使用适配器模式对接多家停车系统

## 状态

已采纳

## 背景

平台当前仅对接 Boot 停车系统，但远期目标是平台化——聚合 Boot、捷顺、科拓等多家停车系统。如果平台业务层直接耦合 Boot API 的报文结构和接口语义，将来每接入一家新系统都需要改动业务代码。

## 决策

在平台领域层与具体停车系统之间引入**适配器（Adapter）**抽象层。

- 定义一套平台内部的停车系统接口（如 `registerVehicle`、`getSpaceCount`、`openGate`）
- 每家停车系统实现一个具体适配器（`BootAdapter`、`JieShunAdapter` 等）
- 平台业务层只依赖抽象接口，不感知底层系统差异

## 后果

- **正面**：接入新停车系统只需新增一个适配器实现，业务逻辑零改动
- **正面**：Boot 特有的概念（orgId、sign 签名、service 名称）被封装在适配器内部，不污染领域层
- **负面**：需要提前设计抽象接口，可能在只有 Boot 一家时显得过度设计
- **缓解**：先基于 Boot API 抽象，等接入第二家时再提炼共性——不需要一开始就完美

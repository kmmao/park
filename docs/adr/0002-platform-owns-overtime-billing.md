# ADR-0002: 超时计费由平台侧负责

## 状态

已采纳

## 背景

租客超出租期仍在场时，Boot 停车系统会在预约车有效期到期后按临时车计费。如果平台同时也收超时费，租客会被双重收费。

## 决策

超时计费完全由平台负责，Boot 侧不产生费用。

实现方式：
- 平台在预约车有效期即将到期时，自动调用 `addvisitorV3` 延长预约有效期，确保 Boot 不触发临停计费
- 平台通过 `inout` 上行推送感知车辆实际离场时间
- 平台根据车场合作方配置的超时计费规则，向租客收取超时费

## 后果

- **正面**：计费逻辑统一在平台侧，不会出现双重收费
- **正面**：超时计费规则可以灵活配置，不受 Boot 临停费率约束
- **负面**：需要后台定时任务可靠地续期预约，如果续期失败会导致 Boot 临停计费
- **缓解**：续期任务需要重试机制和告警

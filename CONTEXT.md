# Boot 停车开放平台 — 领域上下文

## 项目概述

本项目是 **Boot 停车开放平台**（boot-open-api）的第三方对接实现。Boot 平台提供智慧停车场的云端管理能力，第三方系统（如公众号缴费、商场优惠、物业管理）通过本项目与 Boot 平台进行数据交换。

通讯模型分为两个方向：
- **上行接口**（Upstream / Push）：停车系统主动向第三方推送数据（进出场记录、续费记录）
- **下行接口**（Downstream / Pull）：第三方主动调用 Boot 平台查询或操作数据

## 术语表

> 以下术语在代码、接口命名、注释中应保持一致。避免使用括号中标注的同义词。

### 核心实体

| 术语 | 英文标识 | 定义 | 避免使用 |
|------|---------|------|---------|
| 车场 | `park` / `parkId` | 一个独立的停车场实体，由 parkId 唯一标识 | 停车场、parking lot |
| 通道 | `lane` / `laneId` | 车场的物理进出口，分为入口（laneType=1）和出口（laneType=2） | 车道、出入口 |
| 车牌 | `plate` | 完整车牌号（含省份前缀，如"粤B12345"） | license |
| 车牌号（去省份）| `plateNo` | 去除省份前缀的车牌号 | — |
| 进出场记录 | `record` / `recordId` | 一次车辆从入场到出场的完整记录，由平台 recordId 或本地 localId 标识 | 订单、停车记录 |
| 车位 | `space` / `seat` | 物理停车位。`totalSpace` 为总车位，`emptySpace` 为剩余车位，`seatNo` 为具体车位编号 | 泊位 |
| 车库 | `garage` | 车场下的子区域（主车库/子车库），由 `isMainGarage` 区分 | 区域 |

### 车辆分类

| 术语 | `carType` 值 | 定义 |
|------|-------------|------|
| 临时车 | 1 | 无固定车位、按时计费的车辆 |
| 月租车 | 2 | 按月/季/半年/年缴费的固定车辆（登记车的一种） |
| 储值车 | 3 | 预充值余额、按次扣费的固定车辆（登记车的一种） |

> **登记车**（`parkCar`）是月租车和储值车的统称，在系统中通过 `chargeType` 区分具体计费类型。

### 计费类型

| 术语 | `type` 值 | 定义 |
|------|----------|------|
| 按临时车计费 | 1 | 按停车时长计费 |
| 按固定车计费 | 2 | 包月/包季/包半年/包年 |
| 储值车计费 | 3 | 预充值余额扣费 |

> `chargeType`（计费类型 ID）是车场级别的自定义计费方案标识，与上述 `type` 不同。一个车场可配置多种 chargeType（如"月租车""免费车""时段月租车"），每种绑定到一个 type。

### 费用与支付

| 术语 | 字段名 | 定义 |
|------|-------|------|
| 总应收 | `totalFee` | 本次停车产生的总费用（元） |
| 需交费 | `amount` | 扣除已缴和优惠后仍需支付的金额（元） |
| 已缴费用 | `beforePay` | 本次停车中已预缴的金额（元） |
| 现金支付 | `cashFee` | 以现金方式支付的金额（元） |
| 电子支付 | `epayFee` | 以电子方式（微信/支付宝等）支付的金额（元） |
| 临停缴费 | — | 临时车在出场前通过扫码/公众号等方式预缴停车费的业务流程 |

### 优惠减免

| 术语 | 字段/接口 | 定义 |
|------|----------|------|
| 优惠类型 | `discountType` | 1=时间减免（分钟）、2=金额减免（元）、3=折扣、4=全免 |
| 优惠值 | `discountVal` | 具体减免数值，单位取决于 discountType |
| 商家减免 | `DiscountCreate` | 商家为车辆发放停车优惠（如购物满额免停车费） |
| 优惠抵扣查询 | `CarInfoDiscountDestory` | 查询优惠券可抵扣的金额 |
| 取消商家减免 | `DiscountDestory` | 撤销已发放的商家优惠 |

### 预约与业主

| 术语 | 接口 | 定义 |
|------|-----|------|

| 预约车 | `visitor` / `addvisitor` | 业主为来访车辆创建的临时通行预约，包含有效时间段 |

### 黑名单

| 术语 | 接口 | 定义 |

## 接口协议规范

### 认证模型

```
机构（orgId） ──拥有──▶ 机构密钥（orgSecret）
     │
     └── 被授权访问 ──▶ 车场（parkId）
```

- 对接前由 Boot 技术人员分配：**机构 ID**（orgId）、**机构密钥**、**车场 ID**（parkId）
- 每个请求需要携带 `sign` 签名（MD5），签名规则：将公共参数和业务参数分别按 ASCII 排序、`|` 拼接，再拼接机构密钥后取 MD5
- 上行接口（推送）**不需要签名**，下行接口（主动调用）**需要签名**

### 报文结构

**请求（下行）：**
```json
{
  "service": "CarInfo",      // 服务名称（接口标识）
  "version": "01",           // 版本
  "msgId": "uuid",           // 消息唯一 ID
  "orgId": "BOOT01",         // 机构 ID
  "data": { ... },           // 业务参数
  "sign": "md5hash"          // 签名（上行接口无此字段）
}
```

**响应：**
```json
{
  "retCode": 0,              // 0=成功，其他=异常
  "retMsg": "描述",
  "retData": { ... }         // 业务数据
}
```

## 关键业务流程

### 临停缴费流程

```
车辆入场 → 扫码/公众号查询 → 调用 CarInfo 获取费用
    → 用户支付 → 调用 PayNotify 通知平台 → 车辆出场（免费时段内）
```

### 商家优惠减免流程

```
车辆入场 → 商家验证消费 → 调用 CarInfo 查车辆信息
    → 调用 DiscountCreate 发放优惠 → 车辆出场时自动抵扣
```

### 预约车流程

```
业主注册（addOwner） → 创建预约（addvisitorV3）
    → 预约车辆入场自动放行 → 超时未入场可取消（delVisitor）
```
